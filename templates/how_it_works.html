How It Works

Detection Pipeline Explained

Understand exactly how ReviewGuard classifies a review as CG or OR from submission to result.

01

User Submits a Review

A registered user visits the Submit Review page, fills in the product name, category, and pastes or types the review text. A minimum of 20 characters is required. The review is submitted via a secure POST request to the Django backend.

02

Feature Extraction

The detector.py engine extracts multiple linguistic features from the review text. Each feature is scored independently and weighted based on its predictive strength for identifying computer-generated content.

Detection Features Analyzed

Review length (word count)
Generic CG phrase matching
Personal pronoun usage
Exclamation / CAPS abuse
Sentence length variance
Informal language markers
Specific personal details
Time/experience references
03

Scoring & Classification

Feature scores are accumulated and compared against a threshold. A score ≥45 classifies the review as CG (Computer Generated). Below 45 classifies it as OR (Original Review). The confidence percentage is derived from the score distance from the threshold.

04

Result Stored & Displayed

The result (CG/OR), confidence score, and detection reasons are saved to the database linked to the user's account. The result is instantly shown on screen with visual indicators, and becomes visible in both the user's dashboard and the admin panel for monitoring.

05

Admin Review & Override

Administrators can view all submitted reviews in the Admin Review Management panel. They can flag suspicious entries, manually override classifications with justification, and monitor user activity to ensure system integrity.
